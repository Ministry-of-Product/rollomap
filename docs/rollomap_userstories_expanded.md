# RolloMap User Stories

## Founder

- As a Founder I want to ask who in my network would care about a product idea so that I can find early design partners because warm, relevant conversations are more valuable than cold outreach.
- As a Founder I want to see evidence for each recommended person so that I can trust the recommendation because I need to avoid awkward or irrelevant outreach.
- As a Founder I want to filter recommendations by relationship strength so that I can start with people I know well because early feedback is easier with trusted contacts.
- As a Founder I want to filter recommendations by company, industry, or topic so that I can narrow my outreach because different ideas require different audiences.
- As a Founder I want suggested outreach angles for each person so that I can write better messages because generic outreach weakens relationships.
- As a Founder I want to know who introduced me to a person so that I can route outreach appropriately because introductions carry social context.
- As a Founder I want to identify people I have not contacted recently but who are relevant to my idea so that I can revive valuable relationships because networks decay over time.
- As a Founder I want to save a query result as a campaign list so that I can work through follow-ups because product discovery requires organization.
- As a Founder I want to export a shortlist of relevant people so that I can plan outreach in another tool because I may use email, docs, or task systems for execution.
- As a Founder I want to know whether a person has previously expressed interest in a topic so that I can tailor my pitch because relevance creates trust.

## Operator

- As an Operator I want to brief myself before a meeting so that I can remember context because I meet many people across many projects.
- As an Operator I want to see prior interactions with meeting attendees so that I can avoid repeating questions because preparation shows respect.
- As an Operator I want to know open loops with a person so that I can follow through because reliability matters.
- As an Operator I want to search past meeting notes by person and topic so that I can reconstruct decisions because work context is spread across documents.
- As an Operator I want to identify people connected to a project so that I can find the right stakeholder because org memory is often fragmented.
- As an Operator I want relationship context to be separate from company-owned CRM data so that I can preserve my personal network because my relationships move with me.
- As an Operator I want to mark a note as private or sensitive so that it does not appear casually in briefings because some context requires discretion.
- As an Operator I want to correct an incorrect summary so that future answers improve because AI-generated memory must be editable.
- As an Operator I want to know when a person last appeared in my notes or email so that I can understand recency because old context may be stale.
- As an Operator I want to see the source of a claim so that I can verify it because relationship mistakes are costly.

## Professional

- As a Professional I want to remember how I met someone so that I can reconnect naturally because weak ties are easy to forget.
- As a Professional I want to search people by interest so that I can find relevant contacts because opportunities often come from the network I already have.
- As a Professional I want to see a relationship timeline so that I can understand the history because relationships develop over time.
- As a Professional I want to add a manual note after a conversation so that RolloMap captures context because not everything happens in email or docs.
- As a Professional I want to see what I last discussed with someone so that I can continue the relationship because continuity builds trust.
- As a Professional I want to identify neglected relationships so that I can reconnect because important people can fall off my radar.
- As a Professional I want to export my relationship data so that I can take it with me because the data is mine.
- As a Professional I want to delete a person profile so that I can remove data I no longer want because personal control is essential.
- As a Professional I want to know who I know at a company so that I can navigate opportunities because company affiliation is useful context.
- As a Professional I want RolloMap to distinguish facts from guesses so that I can trust the system because AI confidence varies.

## Relationship Builder

- As a Relationship Builder I want briefings that help me care about people better so that I can be more thoughtful because relationships are not transactions.
- As a Relationship Builder I want to remember personal interests someone shared so that I can ask better questions because people feel valued when remembered.
- As a Relationship Builder I want sensitive information to be hidden by default so that I do not misuse it because trust requires restraint.
- As a Relationship Builder I want the system to avoid manipulative language so that I stay relationship-centered because the product should not turn people into targets.
- As a Relationship Builder I want to see conversation starters based on shared history so that I can reconnect warmly because context lowers friction.
- As a Relationship Builder I want to record why someone matters to me so that I can preserve personal meaning because not all value is transactional.
- As a Relationship Builder I want reminders for birthdays or milestones only when I have that data legitimately so that I can reach out appropriately because false intimacy feels creepy.
- As a Relationship Builder I want to tag someone as important to nurture so that the system prioritizes them because some relationships deserve extra attention.
- As a Relationship Builder I want to review AI-generated claims about people so that I can remove anything unfair because my notes may not represent the whole person.
- As a Relationship Builder I want the product to use language like “may be interested” so that uncertainty is clear because people are complex.

## Investor / Advisor

- As an Investor I want to find founders associated with a market thesis so that I can follow up intelligently because deal flow depends on memory.
- As an Investor I want to see when I last spoke with a founder so that I can time outreach well because stale relationships need context.
- As an Investor I want to group people by market, product, or thesis so that I can map my network because investment themes evolve.
- As an Investor I want to know which people have mentioned a company or competitor so that I can understand market signals because conversations contain insight.
- As an Investor I want to review evidence before acting on a recommendation so that I do not overstate knowledge because trust is central to advising.
- As an Advisor I want to identify people who might help a founder so that I can make useful introductions because my value comes from connecting people.
- As an Advisor I want to know intro paths between people so that I can make warm introductions because context improves outcomes.
- As an Advisor I want to track advice previously given so that I can avoid contradicting myself because consistency matters.
- As an Advisor I want to search across notes and email for a portfolio company so that I can reconstruct context because advisory work spans many channels.
- As an Investor I want to keep personal relationship notes separate from firm-owned systems so that I preserve my own context because relationships outlast roles.

## Creator / Community Builder

- As a Creator I want to find people interested in a topic I am writing about so that I can ask for feedback because relevant readers improve the work.
- As a Creator I want to remember who engaged with past ideas so that I can follow up because community forms through repeated interaction.
- As a Community Builder I want to identify clusters of people around themes so that I can organize events or discussions because communities have natural centers.
- As a Community Builder I want to know who might benefit from meeting each other so that I can make introductions because connection is part of my value.
- As a Creator I want to import notes from events so that I can preserve context because event conversations are easy to lose.
- As a Creator I want to tag people by interest area so that I can invite them to relevant conversations because relevance prevents spam.
- As a Community Builder I want to avoid mass outreach based on weak evidence so that I do not damage trust because community requires care.
- As a Creator I want to see evidence-backed audience suggestions so that I can understand why someone is relevant because intuition alone is incomplete.
- As a Community Builder I want to save relationship maps for projects so that I can understand who is connected to what because communities overlap.
- As a Creator I want to keep my network portable so that I do not depend on social platforms because public graphs are performative.

## User / Data Owner

- As a User I want to connect Gmail so that RolloMap can learn from email interactions because much relationship context lives in email.
- As a User I want to connect Google Drive so that RolloMap can learn from meeting notes because my best context is in documents.
- As a User I want to connect Calendar so that RolloMap can understand meetings because attendee history matters.
- As a User I want to upload a LinkedIn export so that RolloMap can seed my contact graph because I already have connection data.
- As a User I want to choose which folders or labels are ingested so that I can limit scope because not all data should be processed.
- As a User I want to pause ingestion so that I can control processing because privacy and cost matter.
- As a User I want to delete a source and derived data so that I can revoke access because ownership requires deletion.
- As a User I want to export my graph so that I can take my data elsewhere because portability is a core promise.
- As a User I want to see ingestion status so that I know what has been processed because opaque background jobs reduce trust.
- As a User I want to revoke OAuth access so that connected accounts are no longer available because permissions must be controllable.

## Privacy-Conscious User

- As a Privacy-Conscious User I want my relationship graph isolated from other users so that no platform-level reputation graph is created because aggregated people intelligence is creepy and dangerous.
- As a Privacy-Conscious User I want no training on my data by default so that my private context is not reused because my relationship data is sensitive.
- As a Privacy-Conscious User I want sensitive claims flagged so that I can review them because personal notes may contain delicate information.
- As a Privacy-Conscious User I want generated claims to cite evidence so that I can judge appropriateness because AI summaries can overreach.
- As a Privacy-Conscious User I want local-first or encrypted storage options so that I can reduce platform exposure because trust increases with control.
- As a Privacy-Conscious User I want clear data deletion behavior so that I know what happens when I delete something because vague privacy claims are not enough.
- As a Privacy-Conscious User I want an audit log of agent access so that I know when my relationship memory was queried because agents need accountability.
- As a Privacy-Conscious User I want to restrict MCP tools by permission scope so that agents cannot access everything because least privilege matters.
- As a Privacy-Conscious User I want to mark people or sources as excluded so that they do not appear in answers because some areas should remain off-limits.
- As a Privacy-Conscious User I want the product to avoid inferring sensitive traits so that it does not create harmful profiles because people deserve dignity.

## AI Agent

- As an AI Agent I want to call a scoped `find_people_for_idea` tool so that I can help the user identify relevant contacts because relationship context improves recommendations.
- As an AI Agent I want to call a `brief_person` tool so that I can prepare the user for meetings because agents should support real-world interactions.
- As an AI Agent I want evidence-backed responses so that I do not hallucinate relationship claims because wrong context can damage trust.
- As an AI Agent I want access only to permitted tools so that I respect user boundaries because agent access must be controlled.
- As an AI Agent I want structured JSON responses so that I can compose reliable workflows because unstructured memory is hard to automate.
- As an AI Agent I want to search interactions by topic and time so that I can answer contextual questions because the user’s memory spans many sources.
- As an AI Agent I want to add a user-approved note so that new context can be saved because memory should improve over time.
- As an AI Agent I want to create an outreach draft with evidence so that the user can review it because sending should remain under user control.
- As an AI Agent I want to list open loops so that I can help the user follow through because commitments are easy to lose.
- As an AI Agent I want access logs created for my calls so that the user can inspect my activity because transparency builds trust.

## System Administrator

- As a System Administrator I want ingestion jobs to be retryable so that temporary failures do not break processing because external APIs are unreliable.
- As a System Administrator I want source sync cursors so that ingestion is incremental because repeated full syncs are inefficient.
- As a System Administrator I want tenant isolation enforced at the data layer so that one workspace cannot access another because privacy is foundational.
- As a System Administrator I want OAuth tokens encrypted so that connector access is protected because tokens are high-risk secrets.
- As a System Administrator I want audit logs for sensitive actions so that incidents can be investigated because trust requires accountability.
- As a System Administrator I want prompt injection defenses in ingestion so that malicious source text cannot control the system because user documents may contain arbitrary content.
- As a System Administrator I want idempotent source processing so that duplicate jobs do not create duplicate people because ingestion must be reliable.
- As a System Administrator I want observability for extraction failures so that quality issues can be diagnosed because AI pipelines fail silently otherwise.
- As a System Administrator I want data retention policies configurable so that privacy commitments can be enforced because deletion must be operationally real.
- As a System Administrator I want connector failures visible to the user so that support burden decreases because users can understand sync problems.

## Product Manager

- As a Product Manager I want to measure successful people-for-idea queries so that I can validate the core value because this is the flagship use case.
- As a Product Manager I want to track user corrections so that I can identify quality gaps because corrections reveal model failures.
- As a Product Manager I want to measure evidence click-through so that I can understand trust behavior because users verify important claims.
- As a Product Manager I want to track duplicate person rates so that I can improve identity resolution because bad merges damage trust.
- As a Product Manager I want to track onboarding completion so that I can improve activation because connectors are required for value.
- As a Product Manager I want to track privacy control usage so that I can understand user concerns because trust drives adoption.
- As a Product Manager I want to see which query types retain users so that I can prioritize development because usage reveals product-market fit.
- As a Product Manager I want to capture discomfort feedback so that I can prevent creepy product behavior because the category is trust-sensitive.
- As a Product Manager I want to compare manual notes versus connector-derived insights so that I can prioritize ingestion sources because each source has different value.
- As a Product Manager I want to test messaging around “private relationship memory” so that the product lands correctly because positioning can make or break adoption.

## Engineer

- As an Engineer I want a clear data model for people, interactions, topics, and evidence so that I can build stable APIs because relationship memory needs structure.
- As an Engineer I want source items separated from generated summaries so that reprocessing is possible because extraction models will change.
- As an Engineer I want embeddings linked to chunks instead of replacing source data so that retrieval remains explainable because vector search alone is not enough.
- As an Engineer I want identity resolution confidence scores so that ambiguous matches can be reviewed because automatic merging is risky.
- As an Engineer I want reversible merge/split operations so that user corrections can recover mistakes because identity errors are inevitable.
- As an Engineer I want structured extraction outputs so that downstream services are predictable because freeform AI output is brittle.
- As an Engineer I want clear MCP tool schemas so that agents can integrate reliably because agent workflows need contract stability.
- As an Engineer I want access control checks in every API path so that workspace isolation is enforced because privacy cannot depend on UI logic.
- As an Engineer I want background job status persisted so that ingestion progress survives restarts because large imports take time.
- As an Engineer I want test fixtures with synthetic relationship data so that privacy-safe automated tests are possible because real user data is sensitive.

## Designer

- As a Designer I want the UI to show evidence next to claims so that users understand why RolloMap says something because trust is visual as well as technical.
- As a Designer I want confidence labels to be human-readable so that users can interpret uncertainty because raw scores are not enough.
- As a Designer I want sensitive data states to be visually distinct so that users slow down before using delicate context because the product must encourage care.
- As a Designer I want the person profile page to feel human rather than clinical so that users do not feel like they are surveilling people because tone matters.
- As a Designer I want merge review flows to be simple so that users can fix identity issues because data hygiene must not feel like work.
- As a Designer I want the Ask interface to provide useful prompt examples so that users discover value because blank boxes can be intimidating.
- As a Designer I want the ingestion status screen to be transparent so that users trust the system because hidden processing creates anxiety.
- As a Designer I want privacy controls to be easy to find so that users feel in control because buried controls reduce trust.
- As a Designer I want recommendation cards to show relationship context so that users can act thoughtfully because names alone are not enough.
- As a Designer I want language throughout the product to avoid surveillance framing so that the product feels respectful because category perception is fragile.
